Month Grid

desktop only
Desktop stacks two months vertically (.cal-2stack > .cal-months-col), each a 7-column grid (.cal-grid) of fixed 46×46px cells (.cal-cell). Mobile reuses the exact same renderMonthGrid() function for a single month, with fluid 1fr columns and aspect-ratio:1 cells instead of the fixed 46px — see the Mobile Month Grid section below. Adjacent-month filler days render dimmed (.other-month, 20% opacity). Today gets a blue cell border + blue day number; past days in the current month fade their day number to 40% opacity.

Day-Count Bubble + Popover

desktop only
.cal-count — a 22×22px circle in the bottom of a day cell showing how many releases land that day. Blue (var(--blue)) normally, amber (var(--amber), .has-pre) if any of that day's games is a pre-order. No border at rest anymore — the old border:2px solid transparent was dead CSS (never overridden anywhere; the hover ring already gives feedback via box-shadow), removed. Click opens #calFloatPop.cal-pop, a singleton position:fixed popover listing each game — each row now carries the same priority-colored left border as the TBA sidebar (.prio-high/medium/low) plus a proper .bdg.b-pre "PRE-ORDER" badge for pre-orders, replacing the old plain amber-text treatment, for visual consistency across all three calendar row types. Live demo below — click a bubble. This is the one calendar element flagged as a deliberate taxonomy exception: it acts like a Chip (standalone, own click action) but its shape (perfect circle, number instead of a text label) doesn't match Chip canon, so per your call it stays a standalone "counter bubble" pattern rather than being folded in.

TBA (Date Unknown) List

different layout per breakpoint
Games with a non-ISO release date (e.g. blank, "TBA", "2027"). On desktop it's a sidebar (.cal-tba-wide, 2-column grid of .cal-tba-chip rows, paginated 20/page with swipe/drag) — now hidden entirely (no reserved space, no "None" placeholder) when there are zero TBA-date games, letting the month grid use the full width. On mobile there's no room for a sidebar, so it collapses into a header toggle — now a proper Filter Pill (.fbar-pill.cal-tba-toggle with a .fbar-pill-count sub-chip, matching every other filter pill's count convention instead of baking "(4)" into the button text) — that swaps the entire body over to the same TBA list full-width, hiding the month view; also hidden when zero TBA games. Each .cal-tba-chip gets a 3px left-border accent from the game's priority (.prio-high=blue, .prio-medium=pink, .prio-low=lime) — confirmed: yes, this reuses the exact same 3 hex values as the Priority pill/badge but through a different mechanism (border-left, not fill), so it's a soft crossover worth checking when you get to the pill/badge color pass. This same left-border pattern has now been extended to the popover and mobile list rows below, for one consistent "row = priority-colored left edge" visual language across all three calendar list surfaces. Open question: chip titles can still silently truncate (2-column grid + ellipsis, no wrap) — a 1-column layout would fix it but roughly doubles the vertical space needed per page; flagging for a decision rather than changing unilaterally.

Mobile Month Grid

mobile only
Originally a flat list grouped by day; replaced entirely with a real single-month grid reusing the exact same renderMonthGrid() function desktop uses, so mobile gets the same day-count bubble + tap-to-open popover interaction instead of a second, different interaction pattern for the same data. .cal-grid/.cal-cell get a mobile override (fluid 1fr columns, aspect-ratio:1) since the desktop fixed-46px sizing assumed the 358px sidebar column width. The old list-view code (.cal-list-item/.cal-list-day etc.) has been deleted entirely, not just hidden.

Out-Today Ticker

identical on both — no mobile CSS override
Lives above the main header (#todayTicker), shown only when something releases today. Fixed pink ribbon label (.ticker-label, solid var(--pink) fill) + a row of clickable plain-text items (.ticker-item, no box — closer to a Stat than anything else, except it's clickable, which Stat by definition isn't). renderTicker() now measures the item row against the visible bar width after render: if it fits, it just sits still (unchanged from before); if it overflows, the content is duplicated once and animated with a linear translateX(0 → -50%) loop — the duplicate makes the loop seamless (the second copy is pixel-identical to the first, so the wrap-around is invisible). Hovering pauses the scroll so items stay clickable. Re-measured on window resize (debounced).

Mobile Month Swipe

mobile only
Swiping the list view left/right changes the visible month (this always existed — it was just invisible: touchend hard-cut to the new month with no feedback during the gesture). Now #calMain live-follows the finger via translateX on touchmove (same drag-tracking mechanic as the card swipe-actions elsewhere in the app), a chevron hint (.cal-swipe-hint-l / .cal-swipe-hint-r, blue gradient) fades in on the side you're dragging toward, and on commit (>50px, horizontal-dominant) the current month slides+fades out while the new one slides+fades in from the opposite edge. Below the threshold it just snaps back. Static preview (opacity forced on) since a drag gesture can't be captured in a screenshot:
